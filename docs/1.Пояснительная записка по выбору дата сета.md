# Пояснительная записка по выбору датасета

## 1. Название и источник датасета

**Customer Churn Dataset — test.csv**  
**Источник:** платформа Kaggle  
**URL:** [Customer Churn Dataset](https://www.kaggle.com/datasets/petetilavanya/customer-churn?select=test.csv)

---

## 2. Описание датасета

Данный датасет содержит информацию о поведении клиентов телекоммуникационной компании. Каждая строка — отдельный клиент, а столбцы отражают активность использования услуг, объёмы звонков, расходы и взаимодействие с технической поддержкой.

---

## 3. Страна и контекст

- Датасет относится к рынку США (стандартный Customer Churn датасет, широко используемый в аналитике и машинном обучении для американских телекомкомпаний).  
- Клиенты в датасете — реальные абоненты телефонной связи.  
- Сервисные данные отражают обычное поведение клиентов массового провайдера услуг связи.  

---

## 4. Период данных

Оригинальные версии Customer Churn собирались за 6–24 месяца.  
В скачанном тестовом файле явного периода нет.  

> Датасет не содержит временной метки, поэтому период сбора данных не определён.  
> Тем не менее, исходный набор Customer Churn традиционно представлял выборку клиентов в течение 1–2 лет.

---

## 📊 Ключевые столбцы

| Столбец                        | Описание |
|--------------------------------|----------|
| account_length                  | Количество дней, сколько клиент является абонентом |
| total_day_minutes               | Сумма минут разговоров в дневное время |
| total_day_charge                | Стоимость дневных звонков |
| total_eve_minutes               | Сумма минут разговоров в вечернее время |
| total_eve_charge                | Стоимость вечерних звонков |
| total_night_minutes             | Сумма минут ночных звонков |
| total_night_charge              | Стоимость ночных звонков |
| total_intl_minutes              | Международные звонки (минуты) |
| total_intl_charge               | Стоимость международных звонков |
| number_customer_service_calls   | Количество обращений в службу поддержки |
| Другие аналогичные показатели  | total_day_calls, total_eve_calls, количество звонков и т.д. |

---

## 🔎 Что показывают эти данные

- **Объём использования услуг:** сколько минут клиент разговаривал в разных режимах.  
- **Стоимость услуг:** показатели `*_charge` показывают оплату за использование услуг.  
- **Обращения в поддержку:** количество обращений (`number_customer_service_calls`) показывает «проблемных» клиентов.  

---

## 📌 Почему этот датасет выбран

Для дипломного проекта «AI-ассистент для 3D-дизайнера» основная цель — анализ поведения потенциальных клиентов и особенности общения.  

Датасет выбран, потому что:

- содержит реальные числовые метрики взаимодействия клиента с сервисом,  
- позволяет демонстрировать навыки анализа данных (описательная статистика, графики, выводы),  
- моделирует поведение клиента — важно для AI-ассистента,  
- подходит для демонстрации методических требований (агрегирование, визуализация, выводы).

---

## 📌 Вывод

Датасет «Customer Churn (test.csv)» — хороший пример структурированных числовых данных.  

- Многомерные числовые характеристики клиентов (минуты, стоимость, обращения).  
- Формализованно показывает профиль поведения пользователей, что важно для анализа клиентов.  
- Наличие данных о количестве обращений позволяет моделировать потенциально «трудных» пользователей — напрямую связано с задачей AI-ассистента.  
- Отсутствие точных временных меток не снижает ценности: анализ выполняется на уровне агрегированных характеристик.
